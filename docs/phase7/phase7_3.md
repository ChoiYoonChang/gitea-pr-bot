# Phase 7.3: 운영 규칙 수립

## 코드 리뷰 프로세스 정의
```markdown
## 팀 코드 리뷰 프로세스
1. PR 생성시 Bot 자동 리뷰 대기 (< 2분)
2. HIGH 이슈는 반드시 수정 후 머지
3. MEDIUM 이슈는 팀 판단으로 수정 여부 결정
4. 인간 리뷰어는 비즈니스 로직에 집중
5. Bot 리뷰가 완료된 후 인간 리뷰 시작
```

## 예외 상황 대응 규칙
- Bot 응답 지연시 대응 방법 (5분 초과시 수동 리뷰)
- 긴급 배포시 프로세스 (핫픽스 브랜치 우선 처리)
- False Positive 신고 절차
